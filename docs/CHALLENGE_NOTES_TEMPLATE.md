# Challenge Notes Template

Use this template to capture key decisions during a 1-day competition.

## Challenge summary
- Briefly describe the prompt and success criteria.

## Users and needs
- Who are the primary users?
- What problems are they solving?

## MVP definition
- What is the smallest, demo-able scope that proves value?

## Chosen features
- List features you are actually building today.
- Mention any trade-offs or shortcuts.

## Future improvements
- Features to add if time permits or after judging.

## Assumptions and constraints
- Tech or platform constraints.
- Data availability assumptions.
- Performance or reliability limits.
